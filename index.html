Application Concept


Hot-or-Not Superhero (Marvel) Edition!

The original concept was to create a multi-player game based on the Marvel and DC Universes. Players would log into the game and select a “tribe” -- Marvel vs DC. Once the tribe was selected – users would pick a Hero or Villain from the respective Superhero Universe. Quickly we pivoted to Marvel only ‘Heroes vs Villains” when we found that DC Comics doesn't publish an API. With this basic premise we conceived the idea of a "Hot versus Not" multi-player game.

The premise of the game: Each player logs in, choses Hero or Villain, and has their hero or villain logged to the system. When the game began the screen would randomly select 2 “Champions” from the pool of all that were selected. In real-time, all play would vote for which hero/villain was Hot – the winner would move onto the next round, the loser, would Not. The scoring was a simple voting system – highest votes wins. After team Squirrelmageddon got started on basic design elements and implementing technologies - we evaluated what would need to be done to make the game work. With the goal of having the whole class to participate – we asked: -What would initiate the start of the game? -What if multiple people all picked the same hero? -Do we wait for everyone to vote – how does the round end? How does the game end?

We wanted to stick with our theme of Marvel, but without a backend we felt managing all the users, choices, game logic, round management etc would be too complex. The decision was made to scale back the game - instead make it more of an information retrieval system for the universe of marvel heroes. This would be done by utilizing the Marvel API and the Wiki API. Users would select a hero or villain, this logs their choice, and they would be able to retrieve additional information on this hero or villain. A nice feature was that users could also see what champions other users selected & query information about those as well!

Design Process

Technologies Used & How They Work

  • HTML - Hypertext Markup Language for web front end development
  • JavaScript - Framework API for client side DOM manipulation
  • CSS - Cascading Stylesheets for dynamic displays
  • Google Firebase - realtime ojbect based API ddriven database
  • jQuery - Javascript frameowrk for dynamic UI
  • Twitter BootStrap - UI framework for responsive design
  • Ajax - jQuery library for API calls
  • JSON - Javascript Object Notation for persistence
  • Moment JS - time management library
  • Popper - tooltip framework to customer experience guidance
  • Modal Dialogs -pop-up displays for errors and warnings
  • SessionStorage - browser storage of JSON objects
  • Hashing - MD5/timestamp for public/private key encoding
  • WebSlides - HTML/CSS presentationnware
